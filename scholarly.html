Human-Computer Interaction homework description

1. Abstract

Your Health is a helpful social tool constructed using web technologies. It was created due to a lack of such a specific platform - dedicated to mental health and everything surrounding it. This document describes how the web application works and other related information.

2. Introduction

Your Health is a tool that I have created as my thesis project, this document serving to show what I have planned for this app and as well as it being my HCI homework. These are the following functionalities that will be implemented :

  • User account creation and login
  • Post creation and publishing with attached tags and other content (such as images and audio clips)
  • Search & filter posts based on tags and users with checkboxes
  • Sort posts based on date / friends / popularity
  • User activity section, with a total per activity and scrollable section to see each activity there is
  • Messaging and adding other users as friends
  • Creating group chats and sorting friends into folders
  • Having a report / flag button for users and posts
  • Text, audio and video calling users / friends
  • A chat settings / customizing button
  • Sorting medical centers based on the distance / rating
  • Filtering medical centers based on the facility type
  • And a lot of other functionalities...

3. Motivation

I chose to create this project out of my personal desire, I always wanted to help everyone I could, if I don't have the skills to become a doctor, then I'll use my IT skills. While there are similar applications such as Reddit , Twitter , Facebook and so on, none of them are specifically dedicated to mental health like this website.

4. Project functionalities

In this part of the document we will present the functionalities of our application and how they work (what files do what and so on).

4.1 Pulse main button

There are a few pulsing main buttons on the base webpage and the prototype webpage, their uses being (for now) : to view the prototype webpage and to return from the legal section.

4.2 Gallery & content

There are a few galleries on the base webpage and prototype webpage where you can click on items to view them in an enlarged format or access other specific content.

4.3 Menu shortcuts

There are menu shortcuts on every page, to ensure easy access.

4.4 Post

The post page is a simple form in which the users can write down and upload contant (images / audio / video).

4.5 Search bars

There are search bars all around the application (user search, forum search, facility search, etc).

4.6 Others

Other functionalities have been previously stated or are (for now) just in the form of wireframes / concepts, thus they haven't been written here due to the large ammount of concepts.

6. Conclusions

Working on this homework, viewing the project from an outsiders pov, thinking about all the issues that they could run into, trying to simulate their questions and needs. All of these things and more have been an interesting experience and challenge, it has really made me open my eyes and appreciate even moreso proper UI.
